# FSP Redesign Memo

FSP (Frustum Surface Probe) の現行設計メモ。

このファイルは `.txt` だが、見出しや箇条書きを使って markdown 風に読む前提で整形する。

---

## 背景

FSP は、SSP より低負荷な補完 GI として、

- **world aligned / clipmap**
- **surface driven allocation**
- **probe pool 上限制御**

を組み合わせた probe system として構成する。

旧案では

- full scan の begin/update
- release list ベースの deferred release
- 一時 flag に依存した debug 状態

などが含まれていたが、現行方針ではそれらを整理し、
**ActiveProbeList 駆動**を中心に再構成した。

---

## 現行方針の要点

### 1. authoritative な動的集合は ActiveProbeList

FSP で「今フレーム有効な probe 集合」は
**`ActiveProbeListCurr` が唯一の正**とする。

- 前フレームの active 集合: `ActiveProbeListPrev`
- 今フレームの active 集合: `ActiveProbeListCurr`

両者は ping-pong で切り替える。

### 2. surface cell と probe は双方向リンク

- cell -> probe:
  - `FspCellProbeIndexBuffer[cell]`
- probe -> cell:
  - `FspProbePoolBuffer[probe].owner_cell_index`

この 2 つが揃って、はじめて割り当て済みとみなす。

### 3. screen-space pass は visible cell の dedup 専用

DepthBuffer を入口にするが、
**screen pass で probe allocation まではやらない**。

screen pass の責務は:

1. world pos を復元
2. FSP cell を求める
3. cell ごとの atomic stamp で dedup
4. `SurfaceProbeCellList` に積む

まで。

### 4. screen-space 以外は indirect dispatch

full-screen / full-pool / full-cell のような広い dispatch を避け、
screen-space pass 以外は次の list から indirect dispatch する。

- `ActiveProbeListPrev`
- `SurfaceProbeCellList`
- `ActiveProbeListCurr`

### 5. 初期実装では non-screen pass の間引きはしない

複雑化を避けるため、初期段階では

- visible cell prepare は list 全件
- active probe update は active 全件
- packed SH update は active 全件
- probe 内 octa texel も全件

の素直な実装を優先する。

## データ契約

### Cell 側

#### `FspCellProbeIndexBuffer`

- 型: `uint`
- 内容:
  - 割り当て probe index
  - 未割り当てなら `k_fsp_invalid_probe_index`

cell 側は **なるべく軽量**にし、
本質的には probe index だけを持つ。

#### `FspProbeBuffer`

cell 側 scratch / debug / legacy mirror 用。

主用途:

- `atomic_work`
  - visible surface cell dedup 用 stamp
- `probe_offset_v3`
  - debug/probe draw 互換の mirror
- `avg_sky_visibility`
  - debug/probe draw 互換の mirror

### Probe 側

#### `FspProbePoolBuffer`

probe 実体の metadata。

最低限必要な内容:

- `owner_cell_index`
- `probe_offset_v3`
- `last_seen_frame`
- `last_update_frame`
- `avg_sky_visibility`
- `debug_last_observed_frame`
- `debug_last_released_frame`
- `flags`
  - 少なくとも allocated 判定用

#### `FspProbeFreeStack`

- `0` 番: stack counter
- `1..N`: free probe index

atomic push/pop 前提。

### Active / visible list

#### `FspActiveProbeListPrev`

- `0` 番: count
- `1..N`: 前フレーム active probe index

#### `FspActiveProbeListCurr`

- `0` 番: count
- `1..N`: 今フレーム active probe index

#### `SurfaceProbeCellList`

- `0` 番: count
- `1..N`: 今フレーム観測された surface cell の global index

### テクスチャ

#### `FspProbeAtlasTex`

- probe ごとの OctahedralMap atlas
- point sample / `Load` 前提
- border copy pass は使わない

#### `FspProbePackedSHTex`

- packed L1 SH atlas
- active probe 更新後に再構築する

## 1 フレームの推奨処理順

### 0. 初期 clear

初期化時のみ:

- cell -> probe を invalid
- probe owner を invalid
- free stack を全 probe で埋める
- active prev/curr count を 0
- visible cell count を 0
- octa atlas / packed SH を clear

### 1. PrevActive indirect arg build

入力:

- `ActiveProbeListPrev`

出力:

- `IndirectArg`

`ActiveProbeListPrev[0]` から
`PrevActiveProcess` 用の indirect dispatch 数を作る。

### 2. PrevActiveProcess

入力:

- `ActiveProbeListPrev`

出力:

- 生存 probe -> `ActiveProbeListCurr`
- 返却 probe -> `FspProbeFreeStack`

各 prev active probe について:

1. `owner_cell_index` を取得
2. owner cell が有効かを見る
3. grid move で invalidate されたかを見る
4. `last_seen_frame` と release grace で stale 判定
5. **継続 active**
   - `ActiveProbeListCurr` に push
6. **release**
   - `cell -> probe` を invalid
   - `probe -> cell` を invalid
   - probe metadata を reset
   - atlas clear
   - free stack に push

現行方針では **release list は持たない**。
release 判定された probe はその場で free stack に返す。

### 3. ObserveVisibleSurfaceCell

入力:

- hardware depth

出力:

- `SurfaceProbeCellList`

screen-space dispatch で:

1. pixel world pos を復元
2. world pos から対象 cascade の FSP cell を求める
3. `FspProbeBuffer[cell].atomic_work` に `frame_count` を書く
4. 同一 frame に初めて観測された cell のみ list に push

この pass は **visible cell dedup 専用**。
ここでは allocation しない。

### 4. VisibleCell indirect arg build

入力:

- `SurfaceProbeCellList`

出力:

- `IndirectArg`

`PrepareVisibleSurfaceCell` 用の dispatch 数を作る。

### 5. PrepareVisibleSurfaceCell

入力:

- `SurfaceProbeCellList`
- `FspCellProbeIndexBuffer`
- `FspProbePoolBuffer`
- `FspProbeFreeStack`
- `FspActiveProbeListCurr`

役割:

- visible cell に既存 probe があれば refresh
- probe 未割り当てなら free から pop して新規割り当て

各 visible cell について:

#### probe あり

- `last_seen_frame` 更新
- `debug_last_observed_frame` 更新
- `owner_cell_index` 整合維持
- probe placement 更新
- `Curr` へ push は不要
  - surviving probe は PrevActiveProcess で積まれているため

#### probe なし

- free stack から pop
- `cell -> probe` を設定
- `probe -> cell` を設定
- `last_seen_frame` 初期化
- `last_update_frame` 初期化
- debug カウンタ初期化
- `Curr` に push

### 6. Active indirect arg build

入力:

- `ActiveProbeListCurr`

出力:

- `IndirectArg`

`UpdateActiveProbe` と `UpdateProbeSh` の両方で使う。

### 7. UpdateActiveProbe

入力:

- `ActiveProbeListCurr`

出力:

- `FspProbeAtlasTex`
- probe metadata

各 active probe について:

1. `owner_cell_index` から cell center を求める
2. `probe_offset_v3` から probe pos を復元
3. octa texel 全件について ray trace
4. radiance / sky visibility を atlas に反映
5. `avg_sky_visibility` を更新
6. `last_update_frame` を更新

### 8. UpdateProbeSh

入力:

- `ActiveProbeListCurr`

出力:

- `FspProbePackedSHTex`

各 active probe について:

1. probe の octa atlas tile を読む
2. L1 SH を積分
3. packed SH atlas の 4 coeff に書く

初期実装では **active probe 全件更新**でよい。
後段で必要なら `DirtyProbeList` を追加する。

## 排他 / atomic の整理

必要な atomic は次だけに絞る。

1. `FspProbeBuffer[cell].atomic_work`
   - visible cell dedup
2. `SurfaceProbeCellList[0]`
   - visible cell list push
3. `FspProbeFreeStack[0]`
   - free push/pop
4. `FspActiveProbeListCurr[0]`
   - active list push

この構造だと、
`PrepareVisibleSurfaceCell` は dedup 済み cell list を処理するため、
基本的に **1 thread / 1 cell** で扱いやすい。

そのため screen-space pass に allocation を押し込むより、

- screen-space: dedup
- visible cell prepare: allocation / refresh

の 2 段に分ける方が安全。

---

## Dispatch 方針

### full-screen dispatch

- `ObserveVisibleSurfaceCell` だけ

### indirect dispatch

- `PrevActiveProcess`
  - 入力: `ActiveProbeListPrev`
- `PrepareVisibleSurfaceCell`
  - 入力: `SurfaceProbeCellList`
- `UpdateActiveProbe`
  - 入力: `ActiveProbeListCurr`
- `UpdateProbeSh`
  - 入力: `ActiveProbeListCurr`

つまり FSP は、screen-space pass を 1 本だけ持ち、
それ以外は **list-driven compute** として組む。

## 現行実装上の注意

### 1. `visible_this_frame` のような一時 flag は不要

debug で「今フレーム観測されたか」を見たいなら:

- `last_seen_frame == frame_count`

で導出する。

### 2. release list は不要

現行方針では

- `release_probe_list`
- `released probe count`
- release 専用 readback

は本筋に不要。

release 判定された probe は
その場で metadata reset + free push でよい。

### 3. debug 用 readback は本筋から切り離す

`free probe count` や `active probe count` などの readback は
debug/stat 用であり、FSP 本筋の成立条件ではない。

### 4. `SrvsParam` の cbuffer packing に注意

member の追加・削除時は
**16-byte 境界**を再確認すること。

特に FSP 領域では:

- int を 1 つ消しただけでも
- 次の `int2` / `float3` / 構造体の位置がずれて
- static_assert に引っかかる

ことがある。

## 初期実装の成立条件

次が満たされていれば、
ActiveProbe 駆動版の骨格として成立している。

1. prev/curr active list でフレームを回せる
2. grid move / stale release が prev active ベースで行える
3. visible surface から probe 新規割り当てができる
4. main update が `ActiveProbeListCurr` indirect で回る
5. packed SH update が `ActiveProbeListCurr` indirect で回る
6. screen-space pass 以外の full scan が除去されている

## 要件説明プロンプト原文

以下は、ActiveProbe リスト駆動化についての要件説明プロンプト原文。
設計整理版ではなく、意図の確認や再読のためにそのまま保持する。

```text
Fspに関するDispatchを最小化し、排他処理も効率化したいので処理フローを整理します。
- データの関係
サーフェイス上のグリッドセル(サーフェイスセル)にはプローブが割り当てられる
グリッドセルにはプローブIndexが格納される
同様にプローブには割り当て先のグリッドセルが格納される
これでActiveプローブがそれぞれどのセルに割り当てられているかわかり
逆にサーフェイスセルにどのプローブが割り当てられているかわかる

Activeプローブリストをうまく構築更新することで、
Activeプローブだけを処理したり、Activeプローブが所属するサーフェイスセルだけを処理するといったことを可能にしたい。


Activeプローブリストは二重化し、効率的に次フレームのActiveプローブリストとFreeプローブリストを構築できるようにする。



- Activeプローブリストの更新
前フレームのActiveプローブリストに対して処理。
Activeプローブの返却猶予フレームやグリッド移動による所属セルInvalidateをチェック。
返却判定となった場合はいろいろリセットしてFreeプローブリストにPushする。
そうでない場合は引き続きActiveなので現フレームのActiveプローブリストにPushする。
これでFreeプローブ返却処理と前フレームからの継続Activeプローブリストができる。
前フレームActiveプローブリストは空になる。


- DepthBufferをガイドにしてサーフェイスセルの準備。
DepthBufferタイルなどのスクリーンスペースで処理。
サーフェイス上のセルにまだプローブが無ければAtomicにプローブ割り当てする。
FreeプローブからPopし、現フレームActiveプローブリストにPush。
セルにActiveプローブインデックスを設定し、Activeプローブ側には所属セルインデックスを設定する。
新規割り当てActiveプローブは返却猶予フレームなどのリセットもする。

- Activeプローブ用のIndirectArg等を作成


- Activeプローブのメイン更新
現フレームActiveプローブリストに対して処理。
Activeプローブについてレイトレなどの更新し、対応するOctahedralMapAtlasを更新。。
返却猶予フレームの更新をする。

- SH更新
現フレームActiveプローブリストに対して処理。
Activeプローブに対応するSHテクスチャ要素を更新。


このようにすることで各シェーダDispatchはそれぞれ最小限のIndirectDispatchができるのではないか。
設計の検討
```
